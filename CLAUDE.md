## AGENT-BASED CODE REVIEW & TESTING STRATEGY

### Принцип: Изолированный контекст для объективных ревью

Для обеспечения качества кода и объективности ревью, ВСЕ проверки кода и тестирование проводятся через отдельных агентов (Agent tool) с изолированным контекстом.

### Агент 1: Code Reviewer (ревью всего кода)
```
Тип: general-purpose
Задача: Полная ревизия кодовой базы
Изоляция: worktree

Проверяет:
- Корректность логики каждого модуля
- Соответствие docstrings и type hints
- Обработку edge cases
- Потенциальные баги (off-by-one, NaN handling, timezone issues)
- Безопасность (нет hardcoded secrets, нет SQL injection)
- Качество API дизайна (правильные возвращаемые типы)
- Воспроизводимость результатов (seed, детерминизм)
```

### Агент 2: Test Writer (TDD для всего кода)
```
Тип: general-purpose
Задача: Написать исчерпывающие тесты для ВСЕЙ кодовой базы
Изоляция: worktree

Покрывает:
1. data_loader.py — загрузка, валидация формата, edge cases
2. preprocessor.py — чистка текста, токенизация, пустые строки, unicode
3. sentiment_analyzer.py — VADER (диапазон, знак, пустые), FinBERT mock
4. price_fetcher.py — загрузка цен, fallback, формат дат
5. correlation_analyzer.py — агрегация, корреляции, Грейнджер, ADF
6. main.py — интеграционный pipeline
7. Новые тесты:
   - test_data_integrity.py — проверка полного датасета (>600K строк, диапазон дат)
   - test_results_integrity.py — проверка всех результатов (диапазоны, NaN, логика)
   - test_pipeline_integration.py — end-to-end на подвыборке
```

### Агент 3: Results Validator (после эксперимента)
```
Тип: general-purpose
Задача: Проверить что все результаты экспериментов логичны и корректны

Проверяет:
- Все корреляции в [-1, 1]
- Все p-values в [0, 1]
- Средний VADER compound близок к 0 (для большой выборки)
- Количество дней в merged = количество уникальных дат
- Количество твитов в день > 0
- Графики не пустые (размер файлов > 0)
- CSV таблицы содержат данные (не только заголовки)
- Согласованность данных между таблицами
```

### Порядок запуска агентов

```
Шаг 1: Agent(Code Reviewer) — ревью кода, выявление проблем
Шаг 2: Исправление найденных проблем
Шаг 3: Agent(Test Writer) — написание полного набора тестов
Шаг 4: Запуск тестов: pytest tests/ -v --cov=src
Шаг 5: Исправление failing tests
Шаг 6: Запуск полного эксперимента на 664K твитов
Шаг 7: Agent(Results Validator) — проверка результатов
```

## ПРОТОКОЛ НАПИСАНИЯ ВКР (АНТИГАЛЛЮЦИНАЦИЯ)

### Стратегия управления контекстом

**Проблема:** 60–70 страниц не помещаются в один контекст.

**Решение — поглавная сборка:**
1. Каждая глава пишется как **отдельная сессия Claude Code**.
2. Перед началом — прочитать CLAUDE.md + контракт данных для главы.
3. Текст главы → markdown-файл `vkr/chapters/XX_name.md`.
4. После написания — верификация всех чисел по исходным CSV.
5. Финальная сборка .docx — отдельная сессия через rudn-vkr-skill.
