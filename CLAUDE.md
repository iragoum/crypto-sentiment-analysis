## AGENT-BASED CODE REVIEW & TESTING STRATEGY

### Принцип: Изолированный контекст для объективных ревью

Все проверки кода и тестирование проводятся через отдельных агентов (Agent tool) с изолированным контекстом. Это защищает от «замыленного глаза» в длинной сессии и от подгонки тестов под уже написанный код.

### Агент 1: Code Reviewer
```
Тип: general-purpose
Задача: Полная ревизия кодовой базы
Изоляция: worktree

Проверяет:
- Корректность логики каждого модуля
- Соответствие docstrings и type hints
- Обработку edge cases (пустые DataFrame, NaN, короткие ряды)
- Потенциальные баги (off-by-one, NaN handling, timezone issues)
- Отсутствие утечки будущего (look-ahead bias) в признаках predictor.py
- Отсутствие hardcoded secrets и API-ключей
- Воспроизводимость (все seed выставлены)
- Отсутствие CUDA-вызовов
```

### Агент 2: Test Writer (TDD для нового кода)
```
Тип: general-purpose
Задача: Написать тесты для новых модулей
Изоляция: worktree

Покрывает (v2):
- power_analysis.py — сверка со статистическими таблицами
- predictor.py — форма X/y, отсутствие утечки, детерминизм, бутстрап CI
- Обновлённые convert_data.py, price_fetcher.py, main.py на кросс-актив
```

### Агент 3: Results Validator
```
Тип: general-purpose
Задача: Проверить корректность всех результатов экспериментов

Проверяет:
- Все корреляции в [-1, 1]
- Все p-values в [0, 1]
- Количество дней в merged = количество уникальных дат
- Количество твитов в день > 0
- Файлы графиков не пустые
- CSV-таблицы содержат данные (не только заголовки)
```
